Insurance Annuity Advisor Agent

A full intake agent that asks qualifying questions, recommends the right annuity, books advisor calls, and sends email summaries — built step by step from zero to production.

Jump to: Use Case Live Demo Build It + Calendar + Email + Web Widget + Teams Adapt It
The Problem This Solves

❌ Without Agent

  • Customer calls or emails asking about annuities
  • Rep manually asks 4–6 follow-up questions
  • Rep looks up matching products
  • Rep does rough income estimate
  • Rep schedules advisor call
  • 20–30 min per inquiry

✅ With Agent

  • Customer types question in chat
  • Agent asks 4 clarifying questions
  • Agent recommends the right product
  • Agent books advisor call automatically
  • Agent emails a summary to customer
  • 3–5 min, no rep needed
ℹ️

Products referenced: Fixed Annuity (guaranteed returns, 3–10 yr horizon, $5K min), Deferred Income Annuity (guaranteed lifetime income, $10K min), Variable Annuity (market-linked, $10K min). Change these to match your actual products.

Fastest path. Use Claude Projects — no API key, no installation, no code. Working agent in 5 minutes.

  1. 1

    Open claude.ai → Projects → New Project

    Name it "Annuity Advisor". Free account works fine.

  2. 2

    Paste these instructions into "Project Instructions"

Agent Instructions — paste into Claude Project
You are an annuity advisor assistant for [Your Company Name].
Help customers find the right annuity product.

Ask these questions ONE AT A TIME (never all at once):
1. How old are they?
2. When do they plan to retire (years from now)?
3. How much do they have to invest?
4. Do they want guaranteed fixed income, or are they open to market risk?

Based on their answers, recommend one of:
- Fixed Annuity: guaranteed returns, low risk, 3-10 year horizon, min $5,000
- Deferred Income Annuity: guaranteed lifetime income starting in 2+ years, min $10,000
- Variable Annuity: market-linked growth, higher potential returns, min $10,000

Always end by asking: "Would you like me to connect you with an advisor?"
Keep responses short (2-4 sentences). Avoid financial jargon.
Never give specific income projections — say "an advisor can calculate your exact figures."

    Test → Share

ℹ️

For developers. ~50 lines of Python. Extends into calendar, email, and web widget in the sections below.

bash — Install
pip install anthropic
python — annuity_agent.py
import anthropic

client = anthropic.Anthropic(api_key="your-api-key-here")

SYSTEM_PROMPT = """
You are an annuity advisor assistant for Meridian Life Insurance.
Ask customers about their age, retirement timeline, investment amount,
and risk tolerance — one question at a time.

Recommend one of:
- Fixed Annuity: guaranteed returns, low risk, 3-10 year horizon
- Deferred Income Annuity: guaranteed lifetime income starting in 2+ years
- Variable Annuity: market-linked growth for risk-tolerant customers

End every recommendation with: "Would you like to schedule a call with an advisor?"
Be warm, concise, avoid jargon.
"""

def run_agent():
    print("Meridian Life Annuity Assistant\n" + "-"*40)
    print("Agent: Hi! I can help you find the right annuity. What's on your mind?\n")

    history = []

    while True:
        user_input = input("You: ").strip()
        if user_input.lower() in ["exit", "quit"]:
            break

        history.append({"role": "user", "content": user_input})

        response = client.messages.create(
            model="claude-opus-4-5",
            max_tokens=500,
            system=SYSTEM_PROMPT,
            messages=history
        )

        reply = response.content[0].text
        history.append({"role": "assistant", "content": reply})
        print(f"\nAgent: {reply}\n")

run_agent()

Book Outlook Calendar Meetings
When a customer says yes, the agent books a 30-minute call in Outlook and sends a calendar invite automatically.
💡

No Azure setup yet? Use Calendly free — create a 30-min "Advisor Call" event type, copy the link, and have the agent include it in its reply when the customer says yes. Zero code, works in 10 minutes.

pip install O365
python — add to annuity_agent.py
from O365 import Account
from datetime import datetime, timedelta

# Your IT admin creates these in Azure Active Directory (10 min setup)
OUTLOOK_CLIENT_ID     = "your-azure-app-client-id"
OUTLOOK_CLIENT_SECRET = "your-azure-app-client-secret"
OUTLOOK_TENANT_ID     = "your-tenant-id"

def book_advisor_call(customer_name: str, customer_email: str, preferred_date: str):
    account = Account(
        (OUTLOOK_CLIENT_ID, OUTLOOK_CLIENT_SECRET),
        auth_flow_type="credentials",
        tenant_id=OUTLOOK_TENANT_ID
    )
    account.authenticate()

    calendar   = account.schedule().get_default_calendar()
    start_time = datetime.strptime(preferred_date, "%Y-%m-%d %H:%M")

    event         = calendar.new_event()
    event.subject = f"Annuity Consultation — {customer_name}"
    event.start   = start_time
    event.end     = start_time + timedelta(minutes=30)
    event.attendees.add(customer_email)
    event.save()

    return f"Booked! 30-min call on {preferred_date}. Invite sent to {customer_email}."
Extension 2
Send Summary Emails
At the end of every conversation, Claude writes a 3-sentence summary and the agent emails it to the customer via Outlook.
python — add to annuity_agent.py
import smtplib
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText

SMTP_SERVER   = "smtp.office365.com"
SMTP_PORT     = 587
SENDER_EMAIL  = "advisor@yourcompany.com"
SENDER_APP_PW = "your-app-password"  # generate at account.microsoft.com → Security → App passwords

def send_summary_email(customer_email: str, customer_name: str, summary: str):
    msg            = MIMEMultipart()
    msg["Subject"] = "Your Annuity Consultation Summary"
    msg["From"]    = SENDER_EMAIL
    msg["To"]      = customer_email

    body = f"""Hi {customer_name},

Thank you for chatting with our Annuity Assistant today.

{summary}

An advisor will be in touch within 1 business day.

Warm regards,
Meridian Life Insurance Team"""

    msg.attach(MIMEText(body, "plain"))

    with smtplib.SMTP(SMTP_SERVER, SMTP_PORT) as s:
        s.starttls()
        s.login(SENDER_EMAIL, SENDER_APP_PW)
        s.sendmail(SENDER_EMAIL, customer_email, msg.as_string())

# At end of conversation: ask Claude to summarize, then send
def email_conversation_summary(client, system_prompt, history, customer_email, customer_name):
    res = client.messages.create(
        model="claude-opus-4-5", max_tokens=150,
        system=system_prompt,
        messages=history + [{"role": "user", "content": "Summarize our conversation in 3 sentences for the customer."}]
    )
    send_summary_email(customer_email, customer_name, res.content[0].text)
Extension 3
Web Chat Widget
A chat bubble on your website. Customers talk to the agent from their browser — no app, no login. A developer deploys this in about 2–3 hours.
bash
pip install flask flask-cors anthropic
python — web_agent.py
from flask import Flask, request, jsonify
from flask_cors import CORS
import anthropic

app    = Flask(__name__)
CORS(app)
client = anthropic.Anthropic(api_key="your-api-key")

SYSTEM_PROMPT = "You are an annuity advisor... [paste your full prompt here]"
sessions = {}

@app.route("/chat", methods=["POST"])
def chat():
    data = request.json
    sid  = data.get("session_id", "default")
    msg  = data.get("message", "")

    sessions.setdefault(sid, []).append({"role": "user", "content": msg})
    res   = client.messages.create(model="claude-opus-4-5", max_tokens=500, system=SYSTEM_PROMPT, messages=sessions[sid])
    reply = res.content[0].text
    sessions[sid].append({"role": "assistant", "content": reply})
    return jsonify({"reply": reply})

if __name__ == "__main__":
    app.run(port=5000)

Deploy to Microsoft Teams (No Code)
Wrap this agent in Copilot Studio and deploy to Teams. Calendar + Email + Chat — all connected, no developer needed. Takes about 1 hour.

    Sign in with your Microsoft 365 work account. Included in your M365 plan — no extra cost.
