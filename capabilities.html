Book Outlook Calendar Meetings
When a customer agrees to a call, the agent books it in Outlook and sends a calendar invite — no rep involved.
ℹ️

What you need: An Azure Active Directory App Registration (your IT admin can create this in 10 minutes). Or use the Calendly alternative below — zero setup.

Install

bash
pip install O365

Add to your agent file

python — calendar_booking.py
from O365 import Account
from datetime import datetime, timedelta

# Set these up once — your IT admin creates these in Azure Active Directory
OUTLOOK_CLIENT_ID     = "your-azure-app-client-id"
OUTLOOK_CLIENT_SECRET = "your-azure-app-client-secret"
OUTLOOK_TENANT_ID     = "your-tenant-id"

def book_advisor_call(customer_name: str, customer_email: str, preferred_date: str):
    """Books a 30-min advisor call in Outlook and emails the customer an invite."""

    account = Account(
        (OUTLOOK_CLIENT_ID, OUTLOOK_CLIENT_SECRET),
        auth_flow_type="credentials",
        tenant_id=OUTLOOK_TENANT_ID
    )
    account.authenticate()

    calendar   = account.schedule().get_default_calendar()
    start_time = datetime.strptime(preferred_date, "%Y-%m-%d %H:%M")
    end_time   = start_time + timedelta(minutes=30)

    event         = calendar.new_event()
    event.subject = f"Annuity Consultation — {customer_name}"
    event.body    = "Follow-up call booked via Fidelity Annuity Assistant."
    event.start   = start_time
    event.end     = end_time
    event.attendees.add(customer_email)
    event.save()

    return f"Done! A 30-minute call has been booked for {preferred_date}. A calendar invite has been sent to {customer_email}."

# Call this when the customer agrees to a meeting:
# result = book_advisor_call("John Smith", "john@email.com", "2026-04-10 14:00")
💡

No Azure setup yet? Simpler alternative: sign up for free at Calendly → create a 30-minute "Advisor Call" event type → copy the link. Have the agent include the link in its reply when the customer says yes. Zero code, works in 10 minutes.

Send Summary Emails After Each Chat
At the end of every conversation, Claude writes a plain-English summary and sends it to the customer's inbox via Outlook.
ℹ️

No extra libraries needed. Python includes email support built-in. You just need your Outlook SMTP settings and an App Password (not your main password — generate one in Microsoft Account security settings).

python — email_summary.py
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

SMTP_SERVER   = "smtp.office365.com"
SMTP_PORT     = 587
SENDER_EMAIL  = "advisor@yourcompany.com"
SENDER_APP_PW = "your-app-password"  # Not your main password — generate at account.microsoft.com

def send_summary_email(customer_email: str, customer_name: str, summary: str):
    """Sends a conversation summary email to the customer."""

    message            = MIMEMultipart("alternative")
    message["Subject"] = "Your Fidelity Annuity Consultation Summary"
    message["From"]    = SENDER_EMAIL
    message["To"]      = customer_email

    email_body = f"""
Hi {customer_name},

Thank you for chatting with our Fidelity Annuity Assistant today.

Here's a summary of our conversation:

{summary}

Next Steps:
- A Fidelity advisor will reach out within 1 business day.
- You can also call us at 1-800-FIDELITY anytime.

Warm regards,
Fidelity Insurance Team
    """

    message.attach(MIMEText(email_body, "plain"))

    with smtplib.SMTP(SMTP_SERVER, SMTP_PORT) as server:
        server.starttls()
        server.login(SENDER_EMAIL, SENDER_APP_PW)
        server.sendmail(SENDER_EMAIL, customer_email, message.as_string())

    return f"Summary emailed to {customer_email}."


# At end of conversation — ask Claude to summarize, then send:
def get_and_send_summary(client, system_prompt, conversation_history, customer_email, customer_name):
    summary_request = conversation_history + [{
        "role": "user",
        "content": "Write a 3-sentence summary of our conversation for the customer's records."
    }]

    summary_response = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=200,
        system=system_prompt,
        messages=summary_request
    )

    summary_text = summary_response.content[0].text
    return send_summary_email(customer_email, customer_name, summary_text)

Deploy as a Web Chat Widget
Put a chat bubble on your company website. Customers talk to the agent directly from their browser — no app, no login.
ℹ️

Two parts: (1) A small Python server that talks to Claude. (2) An HTML snippet you paste into your website. A developer deploys the server to Azure or any cloud hosting in about 2–3 hours.

Part 1 — The Server

bash — Install
pip install flask flask-cors anthropic
python — web_agent.py
from flask import Flask, request, jsonify
from flask_cors import CORS
import anthropic

app    = Flask(__name__)
CORS(app)
client = anthropic.Anthropic(api_key="your-api-key-here")

SYSTEM_PROMPT = """
You are an annuity advisor assistant for Fidelity Insurance.
[Paste your full agent instructions here]
"""

conversations = {}  # In production, use a database like Redis

@app.route("/chat", methods=["POST"])
def chat():
    data       = request.json
    session_id = data.get("session_id", "default")
    user_msg   = data.get("message", "")

    if session_id not in conversations:
        conversations[session_id] = []

    conversations[session_id].append({"role": "user", "content": user_msg})

    response = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=500,
        system=SYSTEM_PROMPT,
        messages=conversations[session_id]
    )

    reply = response.content[0].text
    conversations[session_id].append({"role": "assistant", "content": reply})

    return jsonify({"reply": reply})

if __name__ == "__main__":
    app.run(port=5000)
bash — Run locally to test
python web_agent.py

Part 2 — The Chat Bubble (paste into your website's HTML)

html — paste into your website
<!-- Chat bubble button -->
<button onclick="toggleChat()"
  style="position:fixed; bottom:20px; right:20px; background:#003087;
         color:white; border:none; border-radius:50px; padding:14px 20px;
         font-size:16px; cursor:pointer; z-index:1000;">
  💬 Talk to an Advisor
</button>

<!-- Chat window -->
<div id="chat-window"
  style="display:none; position:fixed; bottom:80px; right:20px;
         width:360px; height:480px; border:1px solid #ccc; border-radius:12px;
         background:white; box-shadow:0 4px 20px rgba(0,0,0,0.15);
         flex-direction:column; z-index:1000;">

  <div style="background:#003087; color:white; padding:14px;
               border-radius:12px 12px 0 0; font-weight:bold;">
    Fidelity Annuity Assistant
  </div>

  <div id="messages"
    style="flex:1; overflow-y:auto; padding:12px; font-size:14px;">
  </div>

  <div style="display:flex; padding:10px; border-top:1px solid #eee;">
    <input id="user-input" type="text" placeholder="Type a message..."
      style="flex:1; padding:8px; border:1px solid #ccc; border-radius:6px;"
      onkeypress="if(event.key==='Enter') sendMessage()" />
    <button onclick="sendMessage()"
      style="margin-left:8px; background:#003087; color:white;
             border:none; border-radius:6px; padding:8px 14px; cursor:pointer;">
      Send
    </button>
  </div>
</div>

<script>
  const sessionId = Math.random().toString(36).substring(7);
  let chatVisible = false;

  function toggleChat() {
    chatVisible = !chatVisible;
    const win = document.getElementById("chat-window");
    win.style.display = chatVisible ? "flex" : "none";
    if (chatVisible && document.getElementById("messages").innerHTML === "") {
      addMessage("agent", "Hi! I can help you find the right annuity. What's on your mind?");
    }
  }

  function addMessage(role, text) {
    const div = document.createElement("div");
    div.style.cssText = `margin:6px 0; padding:8px 12px; border-radius:10px; max-width:85%;
      ${role === "user"
        ? "background:#003087; color:white; margin-left:auto;"
        : "background:#f0f0f0; color:#333;"}`;
    div.innerText = text;
    document.getElementById("messages").appendChild(div);
    document.getElementById("messages").scrollTop = 9999;
  }

  async function sendMessage() {
    const input = document.getElementById("user-input");
    const message = input.value.trim();
    if (!message) return;
    input.value = "";
    addMessage("user", message);

    const res = await fetch("http://localhost:5000/chat", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ session_id: sessionId, message })
    });
    const data = await res.json();
    addMessage("agent", data.reply);
  }
</script>
💡

To go live: A developer deploys web_agent.py to Azure App Service (or any cloud host). Update the fetch URL in the HTML from localhost:5000 to your server's URL. Takes a developer about 2–3 hours total.

Add to Microsoft Teams via Copilot Studio
Deploy your agent inside Teams with zero code. Calendar booking, email sending, and full chat — all connected inside the tool your team already uses.
⚡

This is the zero-code path. Everything below is done through Copilot Studio's visual UI — no developer needed. Takes about 1 hour from start to a working Teams bot.

  1. 1

    Open Copilot Studio

    Go to copilotstudio.microsoft.com and sign in with your Microsoft 365 work account. It's already included — no extra purchase.

  2. 2

    Create a New Agent

    Click "Create" → "New Agent". Describe your agent in plain English:

    Example description (type this into Copilot Studio)
    You are an annuity advisor assistant for Fidelity Insurance.
Ask customers about their age, retirement timeline, investment amount,
and risk tolerance. Recommend the right Fidelity annuity product.
Offer to connect them with an advisor and send a summary email.
  3. 3

    Set Claude as the AI Model (Optional but Recommended)

    Go to Agent Settings → Model → select Claude Sonnet. This gives you Claude's reasoning inside Copilot Studio's deployment infrastructure.

  4. 4

    Connect Outlook Calendar (No Code)

    Go to "Actions" → "Add an action" → search "Office 365 Outlook" → select "Create event". Map the fields: Start Time, End Time, Subject, Attendee Email. Copilot Studio handles the connection automatically.

  5. 5

    Connect Outlook Email (No Code)

    Still in "Actions" → "Add an action" → search "Office 365 Outlook" → select "Send an email". Map: To = customer email, Subject = "Your Consultation Summary", Body = summary from the conversation.

  6. 6

    Deploy to Microsoft Teams

    Go to "Channels" → "Microsoft Teams" → "Turn on Teams" → "Publish". Your agent appears in Teams within a few minutes.

  7. 7

    Share with Your Team

    In Teams, go to Apps → search for your agent by name. Or share the direct link Copilot Studio generates. Pin it to the Teams left rail so everyone sees it every day.
